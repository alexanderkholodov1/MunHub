Connection Guide & Troubleshooting

Quick Start (3 steps)

  1. Plug in the detector via USB — make sure the cable supports data (not charge-only).
  2. Click "Connect" — a popup will show available serial ports. Select your detector.
  3. Click "▶ Record" — data will start flowing to the terminal and Firebase.

Port Picker is Empty? (most common issue)

If you click Connect and the popup shows no ports, your OS is missing the USB-to-Serial driver.

Windows — Install the correct driver:

  • CH340 (most common for Arduino-based detectors):
    Download CH340 Driver
  • CP2102 (Silicon Labs):
    Download CP2102 Driver
  • FTDI:
    Download FTDI Driver

Not sure which chip? Try CH340 first — it's the most common.

After installing:

  1. Unplug and replug the USB cable
  2. Open Device Manager (Win+X) → check that a COM port appears under "Ports (COM & LPT)"
  3. Click "Connect" again in MunHub

macOS: Most drivers are built-in. For CH340: brew install --cask wch-ch34x-usb-serial-driver

Linux: Run ls /dev/ttyACM* /dev/ttyUSB*. If nothing: dmesg | tail -20. Add yourself to dialout: sudo usermod -a -G dialout $USER (then log out/in).

Other Connection Issues

  • "Port is busy" — Close Arduino IDE, PuTTY, minicom, or any other serial monitor using the port.
  • "Permission denied" (Linux) — Run sudo chmod 666 /dev/ttyACM0 or add yourself to dialout group.
  • Wrong browser — Web Serial requires Chrome, Edge, or Opera. Firefox/Safari will use the WebSocket Bridge instead.
  • Data appears but nothing is recorded — Make sure you're logged in and clicked "▶ Record".

Alternative: WebSocket Bridge (any browser)

If Web Serial doesn't work, use the bridge script (works with any browser):

  1. pip3 install pyserial websockets
  2. python3 tools/serial_bridge.py
  3. Click "Connect" — it will auto-detect the bridge
